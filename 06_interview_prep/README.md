# Interview Prep (Review 2 Days Before)

Three files — read them in order, then close the laptop and practice out loud.

## Files

| File | What it covers | When to use |
|---|---|---|
| `flashcards.md` | 30 key terms + definitions | Day before: scan all 30 |
| `system_design_templates.md` | 3 architecture diagrams + blank template | Day before: redraw from memory |
| `common_questions.md` | 15 Q&A pairs | 2 days before: read aloud |

## Day-Before Checklist

- [ ] Can you explain RAG end-to-end without looking?
- [ ] Can you sketch the multi-agent architecture from memory?
- [ ] Can you explain 3 hallucination mitigations?
- [ ] Can you explain why you'd use RAG vs fine-tuning?
- [ ] Can you describe the ReAct loop in 60 seconds?
- [ ] Can you explain how you'd scale the system?
- [ ] Can you explain how you'd debug a failing agent?

## The Frame That Makes You Stand Out

Every answer should mention at least one of:
- **Autonomy** — how does this work across multiple steps?
- **Tool use** — how does this interact with external systems?
- **State/memory** — how is context maintained?
- **Failure handling** — what happens when things go wrong?

The weakest candidates describe happy-path LLM calls.
The strongest candidates immediately ask "what happens when this fails?"
